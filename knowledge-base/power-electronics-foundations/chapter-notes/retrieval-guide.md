# Retrieval Guide

Use this guide when drafting IEEE papers from the power electronics knowledge base.

## Preferred Retrieval Order

1. Start with `concept-map.md` to identify the topic family.
2. Use `domain-vocabulary.md` for precise technical phrasing.
3. Use `paper-language-bank.md` for IEEE-style sentence patterns.
4. Use `chapter-notes/` for chapter-level topic pointers.
5. Use local-only `private-fulltext-cache/` for verification if you have generated it and have the rights to use it.
6. Return to the original PDF for final technical verification when needed.

## Writing Rules

- Use the knowledge base to improve precision, not to fabricate evidence.
- Do not cite the knowledge base as a primary source in a paper.
- Cite the original textbook, paper, standard, or dataset when the claim depends on a source.
- For equations, verify symbols and assumptions against the original source before publication.

## Prompt Examples

```text
Use the power electronics knowledge base to rewrite this converter modeling paragraph in IEEE style.
```

```text
Use the formula index to identify what background I need for a small-signal model section.
```

```text
Use the chapter notes and domain vocabulary to draft a concise introduction paragraph about wide-input-range DC-DC conversion.
```
